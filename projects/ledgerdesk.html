LEDGERDESK

Agentic AI meets financial ops.

SITUATION
  • Financial operations teams spend hours manually triaging transaction exceptions and policy violations
  • Case resolution requires cross-referencing multiple internal tools (transaction timelines, settlement status, account history)
  • Existing automation lacks the reasoning depth to handle nuanced financial edge cases
TASK
  • Build a production-grade agentic copilot that automates transaction exception handling with policy-grounded reasoning
  • Implement human-in-the-loop review with full audit trails for compliance
  • Design a multi-agent pipeline with safety gates and confidence thresholds
ACTION
  • Built multi-agent pipeline: Triage Agent → Retrieval Agent → Tool Planner → Tool Executor → Decision Agent → Safety Gate
  • Frontend in Next.js 15 with a vintage financial design system (cream backgrounds, ledger-style tables)
  • Backend in FastAPI with PostgreSQL + pgvector for RAG over policy documents
  • Redis + Celery for async task queues, full Docker Compose setup
RESULT
  • Functional prototype with case management queue, policy retrieval, and mock internal tool integration
  • Human-in-the-loop workflow: approve, reject, or escalate with full audit trail
  • Designed for production with confidence thresholds and safety gates
  • Roadmap through v2.0 with expanded tool integration

TECH STACK

Next.js FastAPI PostgreSQL pgvector Redis Celery Docker Tailwind CSS
