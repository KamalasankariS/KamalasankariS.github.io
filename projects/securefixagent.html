AUTOMATED VULNERABILITY REPAIR

Patch it. Ship it. Win it.

SITUATION
  • Static analyzers like Bandit flag thousands of potential vulnerabilities but produce many false positives
  • Manual triage and patching is slow, expensive, and doesn't scale
  • Existing LLM-based repair tools lack iterative verification
TASK
  • Build an agentic system that classifies Bandit findings as true/false positives, generates patches, and self-verifies by re-running the analyzer
  • Use only lightweight open-source LLMs (<=8B parameters) to keep inference local and affordable
  • Iterate automatically until the vulnerability is confirmed fixed
ACTION
  • Designed an agentic loop: Bandit scan → LLM classification → patch generation → re-validation (up to 5 iterations)
  • Benchmarked DeepSeek Coder, Qwen 2.5 Coder, CodeGemma, and CodeLlama across classification and repair tasks
  • Integrated inference via HuggingFace Transformers, Ollama, and Apple MLX for flexible deployment
  • Collaborated with Jugal Gajjar on architecture and evaluation
RESULT
  • Published at ICMLA 2025
  • Achieved significant reduction in false positive rates compared to raw Bandit output
  • Demonstrated that small LLMs can perform competitive vulnerability repair when embedded in an agentic verification loop
  • Open-sourced for reproducibility
