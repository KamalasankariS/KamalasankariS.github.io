HYPERCOMPLEX: MULTI-SPACE KNOWLEDGE GRAPH EMBEDDINGS

IEEE BIG DATA · 2025

PROBLEM STATEMENT
  • Knowledge graphs contain diverse relational structures (hierarchical, cyclic, symmetric)
  • Single geometric space embeddings (Euclidean-only) cannot capture all relation types effectively
  • Existing multi-space approaches lack unified optimization frameworks
SOLUTION
  • Simultaneous embedding in Hyperbolic, Complex, and Euclidean spaces
  • Riemannian optimization for geometrically faithful gradient updates
  • Learned gating mechanism to weight contributions from each space per relation type
OUR POSITIONING
  • Adaptive space attention mechanism that dynamically selects optimal geometric representations (hyperbolic, complex, or Euclidean) for each relation type
  • Multi-space consistency regularization loss encouraging agreement between different geometric spaces
  • Scalable architecture with adaptive dimension allocation, scaling from thousands to hundreds of millions of entities
  • Incrementally consolidates and simplifies multi-geometry reasoning through a unified lightweight scoring interface (vs. fixed-geometry methods like TransE, RotatE, ComplEx, SEPA, UltraE)
ARCHITECTURE
  • Triple encoder: Hyperbolic (Poincaré ball), Complex (rotation-based), Euclidean (translation-based)
  • Relation-specific gating network to blend space contributions
  • Riemannian Adam optimizer for manifold-aware parameter updates
  • Scoring function combines distance/similarity from all three spaces
DATASET USED
  • Custom CS Knowledge Graphs from OpenAlex and Semantic Scholar (5 scholarly relation types: AUTHORED, CITES, PUBLISHED IN, BELONGS TO, COLLABORATES WITH)
  • CS-1K: 5,237 entities, 25,317 triples
  • CS-10K: 44,930 entities, 227,502 triples
  • CS-100K: 348,983 entities, 2,162,386 triples
  • CS-1M: 2,384,896 entities, 13,530,177 triples
  • CS-10M: 7,210,506 entities, 44,631,484 triples
  • Standard benchmarks: DBP15K (172,747 entities, 470,781 triples), Hetionet (47,031 entities, 2,250,197 triples), FB15K (14,951 entities, 592,213 triples)
  • All splits: 80% train / 10% validation / 10% test
RESULTS
  • HyperComplEx achieves best MRR across all 8 datasets: 0.831 (CS-1K), 0.771 (CS-10K), 0.704 (CS-100K), 0.608 (CS-1M), 0.612 (CS-10M), 0.411 (DBP15K), 0.642 (Hetionet), 0.692 (FB15K)
  • Up to 5.3% relative gain over ComplEx on CS graphs; 17.5% relative gain on DBP15K
  • Scalability: near-linear (T ∝ |E|^1.06); CS-10M trains in 47.85 hours, inference at 85.23 ms/triple
  • Ablation: removing complex subspace causes largest degradation (CS-1M MRR drops from 0.608 to 0.547)
