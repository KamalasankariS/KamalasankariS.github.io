SECUREFIXAGENT: AUTONOMOUS VULNERABILITY REPAIR VIA LLM + BANDIT OPTIMIZATION

ICMLA · 2025

PROBLEM STATEMENT
  • Static analyzers like Bandit flag thousands of potential vulnerabilities with high false positive rates
  • Manual triage and patching is expensive and doesn't scale
  • Existing LLM-based repair tools generate patches without verification
SOLUTION
  • Agentic loop: Bandit scan → LLM classification (true/false positive) → patch generation → re-validation
  • Multi-armed bandit optimization for exploration-exploitation in repair strategies
  • Uses only lightweight open-source LLMs (<=8B parameters)
OUR POSITIONING
  • Hybrid repair framework integrating Bandit with lightweight local LLMs (<8B parameters) in an iterative detect-repair-validate loop
  • Only system combining Bandit validation, local LLM deployment, AND iterative repair loop (vs. GitHub Copilot, Repairnator, MalCodeAI)
  • All inference performed locally with no cloud reliance, AES-128 encryption at rest, no external telemetry
  • Validation-driven repair mechanism that minimizes hallucinated fixes
ARCHITECTURE
  • Bandit static analyzer as the vulnerability oracle
  • LLM agent for classification, explanation, and patch generation
  • Re-validation loop (up to 5 iterations) re-running Bandit on patched code
  • Multi-armed bandit algorithm to select optimal repair strategies
  • Inference via HuggingFace Transformers, Ollama, or Apple MLX
DATASET USED
  • Synthetic (Injected): 740 samples across 52 vulnerability types with randomized injection locations
  • CVE-Bench: 40 samples across 8 vulnerability types from publicly disclosed CVEs
  • PySecDB: 1,258 samples across 119 vulnerability types from real-world Python security commits
  • SecurityEval: 130 samples across 75 vulnerability types for evaluating code generation
  • Total: 2,168 samples across four sources
RESULTS
  • SecureFixAgent (Fine-Tuned): 87.83% fix accuracy, 8.11% false positives, avg 3 iterations, explanation quality 4.5/5
  • Reduces false positives by 10.8% over static analysis alone
  • Improves fix accuracy by 13.51% over pre-trained LLMs
  • Developer study (n=15): explanation clarity improved from 2.9/5 (raw) to 4.5/5 (fine-tuned)
  • Convergence typically within 3 iterations for fine-tuned models
