MULTILANG CODE PARSER DATASET: UNIFIED AST REPRESENTATIONS ACROSS 10 LANGUAGES

ARXIV · 2025

PROBLEM STATEMENT
  • AI code models train on raw source code without structural understanding
  • No large-scale dataset provides unified AST representations across multiple languages
  • Cross-language code intelligence research lacked a common evaluation schema
SOLUTION
  • Parsed source code from 10 programming languages into a universal AST representation using TreeSitter
  • Standardized schema enables cross-language code analysis on a single dataset
  • Published openly on arXiv and HuggingFace for community use
OUR POSITIONING
  • Provides both hierarchical tree representations and rich metadata for every file, ensuring lossless syntactic coverage and structural uniformity
  • Prior datasets (The Stack, StarCoder, CodeSearchNet) focus on token-level data rather than syntactic or structural representations
  • Universal four-layer AST schema: Metadata Block, Flat Node Array, Node Categorization, Cross-Language Map
  • Four design criteria: Losslessness, Uniformity, Queryability, Scalability
  • Only approach with all four properties (Structural Rep, Cross-Lang, Lossless, Uniformity) vs. existing code datasets and parsing frameworks
ARCHITECTURE
  • TreeSitter-based parsing pipeline for 10 languages
  • Universal AST schema with language-agnostic node types
  • Parquet format for efficient storage and loading (~114 GB total)
  • AST visualization tool for dataset exploration
  • HuggingFace Datasets integration for easy loading
DATASET USED
  • Source data: Permissively licensed source files from the StarCoder dataset (MIT, Apache-2.0, BSD licenses)
  • 10 programming languages: C, C++, C#, Go, Java, JavaScript, Python, Ruby, Scala, TypeScript
  • Total: 7,021,722 parsed files across 40 Parquet files (~114 GB on disk, ~600 GB in memory)
  • Filtering: IQR-based, retained files with 10-10,000 lines; deduplicated via SHA-256 content hashing
  • Preprocessing: UTF-8 normalization, BOM removal, non-printable character removal
  • Parser: Tree-sitter with all 10 grammars compiled into a shared dynamic library
RESULTS
  • 99.99994% AST conversion success rate (4 failures out of 7,021,722 files)
  • Cross-language cosine similarity: C/C++ >0.90; JavaScript/TypeScript 0.96; C#/Scala align with Java at 0.87-0.89
  • PCA projection reveals natural syntactic clusters (Java/Scala, JavaScript/TypeScript as neighbors)
  • Stable ~5.5x compression ratio across all languages
  • Published on arXiv, hosted on HuggingFace and GitHub
