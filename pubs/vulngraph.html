VULNGRAPH: AST/CFG + LLM FUSION FOR VULNERABILITY DETECTION

COMPLEX NETWORKS · 2025

PROBLEM STATEMENT
  • Code vulnerability detection relies heavily on either structural analysis (AST/CFG) or semantic understanding (LLMs), but rarely both
  • Structural methods miss semantic context; LLM-only methods miss control flow patterns
  • Multi-language vulnerability detection (C/C++ and Python) adds complexity
SOLUTION
  • Fused AST + CFG structural features with LLM semantic embeddings
  • Learned gating mechanism to dynamically weight structural vs. semantic signals
  • Unified framework supporting both C/C++ and Python codebases
OUR POSITIONING
  • First framework to apply hybrid network-theoretic graph + lightweight local LLM approach to Java vulnerability detection
  • Most prior GNN work targets C/C++ and Assembly; Java remains underexplored despite widespread production deployment
  • Two-way gating fusion mechanism adaptively combines structural and semantic signals per-sample with interpretable scalar weights
  • Uses lightweight (<4B parameter) local LLMs to avoid cost and privacy concerns of large cloud models
  • Composite training objective combining BCE + InfoNCE contrastive alignment + Laplacian regularization
ARCHITECTURE
  • AST and CFG extraction from source code using language-specific parsers
  • Graph neural network encoding of structural features
  • LLM-based semantic embedding of code snippets
  • Gating network to fuse structural and semantic representations
  • Binary classifier for vulnerable/safe prediction
DATASET USED
  • Vul4J: Reproducible Java vulnerabilities dataset for program repair research
  • Kaggle CVE Fixes: Automated collection of vulnerabilities and fixes from open-source software
  • Kaggle Vulnerability Fixes (Biswas, S.): Vulnerability fix dataset
  • 36,425 code samples collected; 6,000 synthetic safe files created by stripping vulnerable patterns
  • After deduplication and error removal: 35,610 CFG-parsable files (6,234 safe, 29,376 vulnerable)
  • CFGs generated using PROGEX (v3.4.5), averaging 20.6 nodes and 20.3 edges per graph
RESULTS
  • Best overall accuracy: 93.57% (two-way gating fusion)
  • +8.36% improvement over GAT-based embeddings alone (85.21%)
  • +17.81% improvement over Qwen2.5 Coder 3B zero-shot baseline (75.76%)
  • Ablation: removing graphs drops accuracy by 14.64%; removing semantics drops by 8.36%
  • Concatenation fusion: 91.14%, Cross-attention fusion: 92.07%
